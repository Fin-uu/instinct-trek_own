# 🎒 Instinct Trek - AI 智能旅遊規劃助手

> 專為台灣旅遊設計的智能規劃系統，基於 Streamlit 與雙 AI 模型架構。

**Instinct Trek** 是一個全方位的旅遊規劃平台，整合了自然語言處理與生成式 AI 技術。從行程發想到旅途中的即時追蹤，為用戶提供無縫的台灣旅遊體驗。

## 🎯 核心功能 (Core Features)

本系統分為四大核心模組，滿足旅遊的不同階段需求：

### 1. 💬 智能行程規劃 (`pages/Planning.py`)
透過與 AI 的自然對話，快速生成客製化行程。
* **AI 對話助手**：使用自然語言與用戶互動，理解旅遊需求。
* **智能資訊提取**：自動識別關鍵資訊（目的地、天數、預算、人數）。
* **AI 行程生成**：根據需求生成結構化的每日行程安排。
* **台灣知識庫**：整合 `taiwan_knowledge.json`，提供在地化的景點、美食與文化資訊。

### 2. 🗺️ 我的行程管理 (`pages/Mytrip.py`)
視覺化的行程管理與預算控制中心。
* **行程瀏覽**：以時間軸方式展示每日活動安排。
* **預算追蹤**：即時儀表板顯示預算使用狀況（總預算 / 已花費 / 剩餘金額）。
* **費用記錄**：可針對每個單項活動記錄實際花費。
* **彈性編輯**：支援修改行程名稱、日期與預算設定。

### 3. 📍 行程即時追蹤 (`pages/Tracking.py`)
旅途中的個人助理，提供即時資訊與建議。
* **即時動態**：顯示當前及下一個待進行的活動，包含倒數計時功能。
* **環境資訊**：提供未來 5 小時天氣預報與景點人潮模擬（0-100%）。
* **營業狀態**：自動檢查餐廳或景點的營業時間。
* **快捷操作**：一鍵導航、聯絡店家、跳過行程或標記完成。

### 4. ❓ 智能問答 (`pages/Ask.py`)
* 專屬的旅遊諮詢窗口，回答關於台灣旅遊的各類問題（交通、美食、住宿建議）。

## 🤖 AI 技術架構 (Technical Architecture)

本專案採用 **雙模型混合系統 (Hybrid Model System)** 以平衡效能與生成品質：

| 模型組件 | 模型名稱 | 主要用途 | 技術細節與優化 |
| :--- | :--- | :--- | :--- |
| **本地模型** | **vLLM (gpt-oss-120b)** | 資訊提取、<br>語意理解 | • **優勢**：運行速度快，適合處理對話邏輯。<br>• **優化**：針對模型容易產生重複驚嘆號的問題，實作了垃圾回應檢測機制（`count('!')` 驗證）。 |
| **雲端模型** | **Google Gemini 2.0-flash** | 行程生成 (JSON) | • **配置**：Temperature 設為 `0.6` 以提高穩定性；每日 800 tokens 限制。<br>• **容錯**：當 API 呼叫失敗時，自動降級使用 `trip_templates.json` 模板。 |
